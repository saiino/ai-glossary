# Vocab Tracker

AI用語の学習台帳(terms.json)を会話ベースで更新するスキル。
このリポジトリはGitHub Pagesで公開されており、terms.jsonがmainに反映されるとWebページの表示も変わる。

## データ構造(terms.json)

- `meta.levels`: 理解度の定義。0=未学習, 1=なんとなく, 2=理解した, 3=人に説明できる
- `terms[]`: 用語のリスト。各要素は term / description / level / updated を持つ

## できること

### 1. 理解度の更新(メイン機能)

ユーザーの発言から用語と新しい理解度を読み取り、terms.jsonを書き換える。

**発言と理解度の対応の目安:**
- 「なんとなくわかった」「ちょっと読んだ」 → level 1
- 「理解した」「OK」「わかった」 → level 2
- 「人に説明できる」「完全に理解した」「マスターした」 → 3
- 「やっぱりわからない」「忘れた」 → レベルを下げる(発言に応じて0か1)

**手順:**
1. terms.jsonを読み、該当する用語を探す。表記ゆれ(大文字小文字、カタカナ/英語)は柔軟にマッチさせる
2. 該当用語のlevelを更新し、updatedを今日の日付(YYYY-MM-DD)にする。meta.last_updatedも更新する
3. 変更をcommitしてpushする。コミットメッセージは `learn: <用語> を level <旧> → <新> に更新` の形式
4. ユーザーには「◯◯を『理解した』に更新しました(全体: 説明できる2 / 理解した5 / ...)」のように、更新結果と現在のサマリーを短く報告する

**重要な注意:**
- 該当用語が見つからない場合は勝手に追加せず、「◯◯は未登録です。追加しますか?」と確認する
- どの用語か曖昧な場合(部分一致が複数など)は候補を提示して確認する
- ユーザーが明示していない理解度を勝手に高く設定しない。迷ったら本人に聞く

### 2. 用語の追加

「◯◯を追加して」と言われたら、その用語の説明文(2〜3文、初学者にわかる平易さ)を書いてterms.jsonに追加する。levelは指定がなければ0(未学習)。追加後にcommit・push。

### 3. 用語の説明(学習支援)

「◯◯って何?」と聞かれたら、terms.jsonのdescriptionを土台にしつつ、例え話を交えて説明する。説明後に「理解できたらそう言ってもらえれば台帳を更新します」と一言添える。未登録の用語なら説明した上で追加を提案する。

### 4. 学習状況の確認

「進捗どう?」「何覚えた?」と聞かれたら、レベル別の集計と、最近更新された用語、level 0/1で止まっている用語を簡潔にまとめる。次に学ぶと良さそうな用語を1つ提案してもよい。

### 5. 説明文の改善

「◯◯の説明がわからない」「もっとわかりやすく書いて」と言われたら:
1. まず会話の中で、例え話を交えた平易な説明を提示する
2. ユーザーの反応を見て、納得した表現をベースにterms.jsonの
   descriptionを書き換える(2〜3文、初学者にわかる平易さを維持)
3. ユーザーが具体的な直し方を指定した場合(「◯◯の例えで書いて」など)は
   それに従う
4. commit メッセージは `docs: <用語> の説明文を改善` の形式

## Git操作の注意

- 変更対象はterms.jsonのみ。index.htmlやこのSKILL.mdは、ユーザーが明示的に頼んだ場合以外は変更しない
- クラウド実行(Claude Code on the web)の場合、pushはブランチ+PRになる。その場合は「PRを作成しました。マージするとページに反映されます」とマージが必要なことを必ず伝える
- JSONの構文を壊さないこと。書き換え後、可能ならJSONとしてパースできることを確認してからcommitする
