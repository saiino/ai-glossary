# ai-glossary v2 改修指示書

このリポジトリ(AI用語学習トラッカー)に以下の3つの改修を行ってください。
作業前に現状の terms.json / index.html / .claude/skills/vocab-tracker/SKILL.md を読んで構造を把握すること。
既存の機能(習熟度ヒートストリップ、レベル別フィルタ、検索、レベルチップの色分け)は壊さず維持してください。

## 改修1: terms.json に関連語フィールドを追加

各用語に `related` フィールド(文字列の配列)を追加する。値は**既存用語の term と完全一致する文字列**とする。

```json
{
  "term": "RAG",
  "description": "...",
  "related": ["埋め込み(Embedding)", "ハルシネーション"],
  "level": 1,
  "updated": "2026-07-12"
}
```

- 既存10用語すべてに、意味的に関連する用語を related として設定する(関連がなければ空配列でよい)
- 関連は原則**双方向**にする(AのrelatedにBがあれば、BのrelatedにもAを入れる)
- 存在しない用語名を related に入れないこと

## 改修2: index.html をモーダル(ポップアップ)方式に変更

### 一覧のスリム化
- 現在のカードから説明文(term-desc)を外し、「用語名 + レベルチップ」のコンパクトな行にする
- 一覧の行をタップするとモーダルが開く

### モーダルの内容
- 用語名、レベルチップ
- 説明文(全文)
- 関連語をタグ(チップ)として表示。**関連語タグをタップするとその用語のモーダルに切り替わる**(用語間を回遊できるように)。related が空なら関連語セクションごと非表示
- 更新日
- 閉じる方法: ×ボタン、モーダル外タップ、Escキーの3通り

### 品質要件
- スマホ(幅380px程度)での操作を最優先。モーダルは画面下からのシートでも中央ダイアログでもよいが、片手で閉じやすいこと
- モーダル表示中は背景スクロールを止める
- 既存のデザイントークン(CSS変数、配色、フォント)を踏襲し、見た目の一貫性を保つ
- ヒートストリップのマスをタップした場合も、該当用語のモーダルを直接開く動作に変更してよい(スクロールジャンプより自然なため)

## 改修3: SKILL.md の機能追加

vocab-tracker の SKILL.md に以下の2機能を追記する。既存の記述スタイル(手順・注意点を「なぜそうするか」とともに書く)に合わせること。

### 3-1. 説明文の改善

「◯◯の説明がわからない」「もっとわかりやすく書いて」と言われたら:
1. まず会話の中で、例え話を交えた平易な説明を提示する
2. ユーザーが納得した表現をベースに terms.json の description を書き換える(2〜3文、初学者にわかる平易さを維持)
3. いきなりファイルを書き換えず、必ず会話で合意してから反映する(納得していない説明で上書きしても学習にならないため)
4. コミットメッセージは `docs: <用語> の説明文を改善`

### 3-2. 関連語の管理

- 用語を新規追加するときは、既存用語から関連しそうなものを related に設定し、**相手側の related にも追加**して双方向を保つ
- related に入れられるのは既存用語の term と完全一致する文字列のみ。存在しない用語を入れない
- 「◯◯と◯◯は関連づけて」と言われたら双方向に追加する
